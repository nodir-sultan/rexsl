02-Mar-2014 0.12

rexsl:check

Full name:

com.rexsl:rexsl-maven-plugin:0.12:check

Description:

Test the entire project against RESTful principles.

Attributes:

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.
  • Binds by default to the lifecycle phase: integration-test.

Optional Parameters

Name Type Since Description
check String 0.3.6 Name of the check class to execute (if not set - all checks are executed).

Set it to some irrelevant value in order to see a full list of available checks, for example: mvn rexsl:check -Drexsl.check=foo.


User property is: rexsl.check.
port Integer - TPC port to bind to (by default a random port is used).
User property is: rexsl.port.
skip boolean - Shall we skip execution?
Default value is: false.
User property is: rexsl.skip.
skipTests boolean 0.3.8 Skip all tests.
Default value is: false.
User property is: skipTests.
systemPropertyVariables Map 0.3 System variables to be set before running tests.

You define them in a similar way as in maven-surefire-plugin, for example you want to reconfigure LOG4J just for the tests:

<plugin>
  <groupId>com.rexsl</groupId>
  <artifactId>rexsl-maven-plugin</artifactId>
  <configuration>
    <systemPropertyVariables>
      <log4j.configuration>./x.props</log4j.configuration>
    </systemPropertyVariables>
  </configuration>
</plugin>

test String 0.3.6 Regular expression that determines tests (groovy, xml, etc.) to be executed during test.
Default value is: .*.
User property is: rexsl.test.
webappDirectory String - Webapp directory.
Default value is: ${project.build.directory}/${project.build.finalName}.
User property is: rexsl.webappDirectory.
