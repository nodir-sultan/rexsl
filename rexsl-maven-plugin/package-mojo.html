rexsl:package

Full name:

com.rexsl:rexsl-maven-plugin:0.12:package

Description:

Package resources.

Text resources should be packed before they got delivered to the WAR package. For example, comments should be removed from CSS and XSL stylesheets, JavaScripts should be minified, etc. We have a number of convenient packers that do this job, you just need to call them. First of a package goal should be called:

<plugin>
  <groupId>com.rexsl</groupId>
  <artifactId>rexsl-maven-plugin</artifactId>
  <version>...</version>
  <executions>
    <execution>
      <phase><phase>
      <goals>
        <goal>package</goal>
      </goals>
    </execution>
  </executions>
</plugin>

The goal will package all files it knows about and place their packaged (minified) versions into webappDirectory. Then, you should configure maven-war-plugin to not touch these files again (to avoid their packaging):

<plugin>
  <artifactId>maven-war-plugin</artifactId>
  <configuration>
    <warSourceExcludes>js/**, css/**, xsl/**</warSourceExcludes>
    [..]
  </configuration>
</plugin>

Attributes:

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.
  • Binds by default to the lifecycle phase: process-resources.

Optional Parameters

Name Type Since Description
filtering boolean 0.3.7 Static resources (CSS, XSL, JS, etc.) should be filtered before packaging.

At the moment only properties defined in pom/properties section are supported.


User property is: rexsl.filtering.
port Integer - TPC port to bind to (by default a random port is used).
User property is: rexsl.port.
skip boolean - Shall we skip execution?
Default value is: false.
User property is: rexsl.skip.
systemPropertyVariables Map 0.3 System variables to be set before running tests.

You define them in a similar way as in maven-surefire-plugin, for example you want to reconfigure LOG4J just for the tests:

<plugin>
  <groupId>com.rexsl</groupId>
  <artifactId>rexsl-maven-plugin</artifactId>
  <configuration>
    <systemPropertyVariables>
      <log4j.configuration>./x.props</log4j.configuration>
    </systemPropertyVariables>
  </configuration>
</plugin>

webappDirectory String - Webapp directory.
Default value is: ${project.build.directory}/${project.build.finalName}.
User property is: rexsl.webappDirectory.
